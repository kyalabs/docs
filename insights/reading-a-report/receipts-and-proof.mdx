Every number in a delivered report is a link.

Click any count, percentage, or finding, and the **proof view** opens — the receipts behind that specific number. The appendix _is_ the receipts; there is no separate "evidence section" you have to cross-reference.

## What a receipt contains

A receipt is one trip's complete record. It contains:

- **The agent** — which model, which version, which retrieval mode.
- **The persona cohort** — the shopper profile the trip ran as.
- **The mission** — the exact question the agent was asked.
- **The steps** — what the agent did, in order: searches, page fetches, source citations, intermediate reasoning where the agent exposes it.
- **The sources** — every URL the agent touched, with access posture: **reached**, **access-limited**, or **insufficient**.
- **The artifact** — the agent's final answer, captured verbatim.
- **The classification** — how this trip was scored against the metric (e.g., _surfaced / not surfaced_).

A receipt is a queryable record, not a screenshot. You can open it in the platform and link to it today; structured export is rolling out (see [What's exportable](#whats-exportable)).

## The proof view

The proof view is the platform UI for receipts. It opens to one number, then expands to all the trips that compute that number.

<Steps>
  <Step title="Click a number in the report.">
    Any count, percentage, or finding. The proof view opens to it.
  </Step>
  <Step title="Read the trip list.">
    Every trip that contributes to the number, with its classification and access posture.
  </Step>
  <Step title="Open a single trip.">
    The full receipt — agent, persona, mission, steps, sources, artifact, classification.
  </Step>
  <Step title="Open the static proof artifact.">
    The proof view is a static `proof_report.html` artifact alongside the receipts list — the captured record behind that number, not a live-filtering UI.
  </Step>
</Steps>

## Why this matters

The proof view is what makes the verdict defensible. If someone challenges the headline in a meeting, you don't argue methodology — you open a receipt.

It's also what separates the rig from a scraper dashboard: a scraper gives you a mention count; the proof view gives you the trip behind every count. You don't have to trust the number; you can check it.

## What's exportable

| Object | Status |
| --- | --- |
| The full receipt for one trip | Rolling out — JSON export, includes the agent's verbatim artifact. Today the read path is an operator-admin retrieval, not self-serve. |
| The receipt set behind one finding | Rolling out — bulk export with cloud-storage handoff, on the same self-serve track. |
| The proof view itself (as a shared link) | Yes — shareable via an unguessable link to a specific number. |

<Note>
  Per-trip JSON and bulk receipt export are **rolling out**: built, but served today through an operator-admin read path rather than an in-product self-serve action. Until that lands, ask us and we'll pull the export for you.
</Note>
