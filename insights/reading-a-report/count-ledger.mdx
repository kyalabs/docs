The count ledger is the math behind the verdict, on the page.

Most research artifacts hide their denominator. The count ledger puts it up front, because a number without its denominator is a vibe. When a study says you appeared in 40% of trips, you always see **40% of what** — so the number survives scrutiny in the room.

## The five counts

Every delivered report carries these five counts, computed across the panel for the mission:

| Count | What it is |
| --- | --- |
| **Planned** | How many trips the study was designed to run — panel size × personas × mission shape. The intended denominator. |
| **Completed** | Trips that ran end to end and produced a capturable artifact. |
| **Did not run** | Trips that never completed a run — the cell couldn't get through the pipeline, so no artifact was produced. Incomplete, not a result. |
| **Insufficient** | Trips that ran but produced no usable output — the capture didn't yield enough to compute a finding. |
| **Metric-eligible** | The count that actually feeds the verdict — completed trips that meet the metric's criteria. The honest denominator. |

These add up. `planned = completed + did-not-run + insufficient`, and `metric-eligible ≤ completed`. The arithmetic is on the page so you can check it.

## Why the denominator is a first-class fact

The same finding can be a strong result or a weak one depending on which count you compare it to. _"You appeared in 40% of trips"_ means something very different against 19 completed trips than against 25 planned trips where several did not run.

The count ledger makes that distinction structural rather than rhetorical. The report **must** state which denominator the verdict rests on — typically metric-eligible — and the verdict's confidence is bound to it.

<Note>
  This is part of [the Honesty Layer](/evidence/honesty-layer): the report cannot compute a percentage without naming the denominator it computed it from. The denominator is a rendered field, not a footnote.
</Note>

## A worked example

A study runs a category question against a 25-trip plan:

| Count | Value |
| --- | --- |
| Planned | 25 |
| Completed | 19 |
| Did not run | 4 |
| Insufficient | 2 |
| Metric-eligible | 19 |

The verdict says _"Surfaced in 8 of 19 metric-eligible trips (42%)."_ Not _"42% of the time."_ The denominator travels with the number.

The 18 trips that did not run are incomplete cells, not a result — they're held out of the denominator rather than counted against you. Reach, where an agent got to a surface but was access-limited, is a separate per-finding signal covered on its own page.
