ShoppiPanel is a versioned instrument, with every number bound to a receipt. Everything below is observable in a delivered study, down to the trip behind each count.

<Note>
  This section is for your **data and engineering evaluators**: the method, the schema, and the data-handling posture. If you're here for what ShoppiPanel does and how to read a study, [Evaluating the insights](/insights/overview) is your section.
</Note>

## The rig in one frame

A study runs in five stages, in this order, every time:

<Steps>
  <Step title="Question: the mission a real shopper would ask">
    A category question, scoped to a brand set and a surface set. Pinned to a panel version so reads stay comparable.
  </Step>
  <Step title="Panel: the versioned instrument">
    A configured set of agents, model versions, and persona cohorts (the shopper profiles a mission runs as). Pinned, signed, and reproducible.
  </Step>
  <Step title="Runs: agents shop the live web">
    Each agent in the panel runs the mission against the live web, in its native retrieval mode. Browse-native and search-native are captured as [separate lanes, resolved to one denominator](/how-it-works/two-lanes).
  </Step>
  <Step title="Evidence: the receipt for every trip">
    Every trip is captured end to end: the question, the steps the agent took, the sources it touched, what it could and couldn't reach, the artifact it produced. Stored as queryable records.
  </Step>
  <Step title="Report: bound to the evidence">
    The read is computed from the data; the data is computed from the evidence. The headline cannot assert a result the counts don't support. See [the Honesty Layer](/evidence/honesty-layer).
  </Step>
</Steps>

## What "every number opens to a receipt" means, concretely

Every count, every finding, every claim in a delivered report carries a pointer to the trips behind it. Open the pointer and you see the agent, the persona cohort, the steps it took, the URLs it touched, the access posture of each one (reached, access-limited, insufficient), and the artifact the agent produced.

The appendix _is_ the receipts, not a summary of them. A number you can't walk back to a trip doesn't ship.
